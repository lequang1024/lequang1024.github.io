Deep Link QR Code Generator Documentation

This document outlines the functionality and usage guidelines for the Deep Link QR Code Generator tool, designed to enhance deep link creation and testing workflows.

Overview

The Deep Link QR Code Generator is a client-side web application that simplifies the construction and testing of game-related deep links. It enables developers and QA personnel to dynamically generate complex URLs, produce real-time scannable QR codes, and initiate deep link testing directly on mobile devices. All data is stored locally within the browser to ensure persistence across sessions.

Key Features

  • Dynamic URL Generation: The URL updates automatically as parameters are modified.
  • Environment Selection: Easily switch between predefined environments (Trunk, Trunk Liveops, QA3) or input a custom backend URL.
  • PR Number Support: Automatically formats the backend host using the specified pull request number.
  • Customizable Query Parameters: Users can enable and configure a wide range of optional parameters to tailor deep links for specific testing needs.
  • Live QR Code Rendering: The QR code updates in real-time as the link configuration changes.
  • Sharable Configuration Links: Generate a unique URL that retains the current configuration for easy collaboration.
  • Server Availability Check: Automatically validates whether the target backend is reachable.
  • Grafana Log Integration: Direct link support for accessing relevant logs on Grafana.
  • Persistent Storage: Previous configurations are saved locally for future reuse.

How to Use

Base Configuration

  1. Select an Environment: Choose the desired environment from the dropdown menu.
  2. PR Number: For environments such as Trunk or QA3, provide a pull request number to construct the target host address.
  3. Custom Backend URL: Optionally, input a custom backend address for testing purposes.

Additional Query Parameters

Enable optional parameters as needed. These are intended for advanced testing scenarios.

  • Check the box next to each parameter to include it in the generated deep link.
  • Enter appropriate values in the input fields provided.
  • The generated link and QR code will reflect changes in real-time.

Alive Check

The application performs a reachability check on the backend server after input is provided.

Note on CORS: The reachability test uses a no-cors request. While it cannot confirm the specific nature of the endpoint response due to browser security restrictions, it can determine general accessibility from the client device.

Available Actions

  • Start Game: Attempts to open the generated deep link on a mobile device.
  • Copy Sharable Link: Copies a URL to the clipboard that includes all current configurations.
  • Reset All Settings: Clears all configurations and resets the interface to default values.
